Software Livre

Como usuários de diversos softwares, devemos estar cientes que eles podem ser classificados quanto ao tipo de sua licença como:

  • Comercial liberados para testes.
  • Shareware (Liberado para testes por um período).
  • Freeware (Software gratuito).
  • Liberado para uso não comercial.

    • Gnu Public License (GPL)

  • Copyleft:
    • Mozilla Public License (MPL)

As licenças citadas acima são as mais comuns, há inúmeras outras como as listadas no artigo "Várias Licenças e Comentários sobre elas" da "Free Software Foutation" http://www.gnu.org/licenses/license-list.pt-br.html. Mas estamos mais interessados em esclarecer os conceitos de software livre e software gratuito. Basicamente um software gratuito (freeware), é todo aquele que você usar sem precisar pagar nada por ele.

FREE SOFTWARE FOUDATION. Várias Licenças e Comentários sobre elas. Web. Disponível em: http://www.gnu.org/licenses/license-list.pt-br.html.

Já o conceito de software livre é mais complexo, e segundo os critérios da http://www.gnu.org/home.pt-br.html Free Software Foundation (FSF) ou em português Fundação Para o Software Livre, é uma questão de liberdade e não de preço. De acordo com a FSF o termo Software livre refere-se à liberdade dos seus usuários poderem executar, copiar, distribuir, estudar, modificar e aperfeiçoar o software de acordo com suas necessidades, sem precisar de dar satisfação a ninguém. Basicamente este termo refere-se a quatro tipos de liberdade, para os usuários do software, ou seja, a liberdade de

  1. executar o programa, para qualquer propósito (liberdade nº 0);
  2. estudar como o programa funciona, e adaptá-lo às suas necessidades (liberdade nº 1). O acesso ao código-fonte é um pré-requisito para esta liberdade;
  3. redistribuir cópias de modo que você possa ajudar ao seu próximo (liberdade nº 2);
  4. aperfeiçoar o programa, e liberar os seus aperfeiçoamentos, de modo que toda a comunidade se beneficie (liberdade nº 3). O acesso ao código-fonte é um pré-requisito para esta liberdade.

Portanto, de acordo com a FSF, um programa é considerado um software livre se os seus usuários tem todas essas liberdades. Note que esse termos não impede que você possa vender o software, entretanto, você deve distribuir sem custo algum o código fonte do software. Portanto, acesso ao código-fonte é uma condição necessária ao software livre.

Para que as liberdades listadas acima sejam reais, elas devem ser irrevogáveis desde que você não faça nada errado; caso o desenvolvedor do software tenha o poder de revogar a licença, mesmo que você não tenha dado motivo, o software não é livre.

Em 1984, para implementar essas ideias Richard M. Stallman criou a "Free Software Foundation" e lançou o projecto GNU. A licença do projeto GNU, a Licença Pública Geral GNU (GNU General Public License, GNU GPL ou GPL simplesmente quando o contexto não permitir dúvidas sobre ao que se refere), não somente concede as quatro liberdades descritas acima, mas também as protege. Graças a essa proteção, a GPL é, hoje em dia, a licença mais utilizada para o Software Livre.

Ao lado da GPL existem outras licenças que concedem essas liberdades, o que as qualifica de licenças de Software Livre. Uma delas, a licença FreeBSD, merece uma menção particular. A principal diferença com a GPL é que ela não procura proteger a liberdade.

Quando se fala de Software Livre, uma confusão frequente é de pensar que um tal software deve ser grátis (principalmente porque em inglês Free significa livre, mas também significa grátis). Na realidade, uma grande parte dos protagonistas do Software Livre trabalham no campo do Software Comercial.

Em 1998, o norte-americano (E.U.A.) Bruce Perens escreveu uma definição para o "Open Source" (Open Source Definition), a qual descrevia as propriedades técnicas do Software Livre e foi utilizada como texto fundador do movimento "Open Source" (Open Source Movement).

A "Definição do Open Source" é ela mesma derivada das "Linhas Diretrizes do Software Livre Debian", que derivam das quatro liberdades mencionadas acima. Consequentemente, as três definições descrevem as mesmas licenças; a "Licença Pública Geral GNU" (GPL) é a licença de base de todas as definições.

O movimento "Open Source" tem por objetivo ser um programa de marketing do Software Livre. Esse objetivo deliberadamente ignora todos os aspectos filosóficos e/ou políticos do software livre, pois estes aspectos são considerados prejudiciais à comercialização.

Por outro lado, o movimento Software Livre considera o ambiente filosófico/ético e político como uma parte essencial do movimento e um dos seus pilares fundamentais.

Editores e Processadores de Texto

Boa parte do trabalho realizado pelos usuários de computadores, concentra-se na edição de textos, os quais podem: um texto formatado, um programa, um arquivo de configuração que é um texto sem formatação, um texto para internet, script, etc. Aqui a formatação refere-se ao formato do texto, com caracteres especiais, fontes específicas em cada parte do mesmo, cores, etc. Um texto sem formatação ou texto plano, refere-se a um arquivo que pode conter alguma formatação dada por espaços e pelo caractere tabulação. Os programas, os arquivos de configuração, etc, são todos arquivos sem formatação ou textos planos. Portanto, para editarmos esses textos temos os processadores de textos que editam os textos formatados e os editores de textos usamos para referirmos a textos não formatados. Por exemplo, no Microsoft Windows o Word é um processador de texto e o Notepad é um editor de texto, e para o GNU/Linux o OpenOffice.org, kOffice, o AbiWord, etc são exemplos de processadores de textos enquanto o vi, o emacs, o nano, o mcedit, o gedit, o kate, etc. são exemplos de editores de textos.

Para escrevermos um texto devemos saber se ele é

  1. um texto puro, sem formatação, por exemplo,
    • um arquivo de configuração;
    • um programa;
    • um arquivo de dados.
    • etc.
  2. um texto com formatação, por exemplo,
    • um artigo;
    • um livro;
    • uma tese;
    • uma monografia;
    • etc.

O que chamamos de pacote de escritório ou mesmo uma suíte para escritório é um conjunto de softwares constituído por um processador de texto, uma planilha eletrônica, um criador de apresentações (slides) e um banco de dados. Imagens

Ao escrevermos um texto qualquer, eventualmente precisamos de uma imagem para ilustrar ou descrever melhor aquilo que estamos tratando. Nesses casos é conveniente usar um software adequado para desenharmos ou tratarmos uma imagem prévia. Mas antes de escolhermos o software é necessário sabermos qual será o tipo da imagem que iremos tratar.

Uma imagem digital é uma representação bidimensional de uma imagem por conjunto finito de valores digitais, chamados pixeis. Ela é representada por uma matriz 2 por 2, em que elemento da matriz é um pixel. Portanto, para o computador uma imagem é uma matriz $N_{linhas}×N_{Colunas}$ a qual a cada elemento da matriz associa-se um número, o qual é denominado nível de cinza e representa algumas propriedade do pixel tais como cor, tonalidade, brilho, etc. O valor atribuído ao pixel corresponde basicamente a sua cor, e essa costuma ser representada por percentagem de 3 cores: vermelho, verde e azul, o conhecido RGB. Por exemplo: 20% de vermelho + 10% de verde + 5% azul.

Quanto maior o número de linhas e colunas da matriz, ou seja, o número de elementos da matriz melhor será a qualidade da imagem. Quanto maior a número de informações que conseguirmos guardar em cada pixel (aqui, podemos pensar que cada pixel é um vetor, e nesse caso temos uma matriz tridimensional), maior será a quantidade de cores que poderemos colocar em cada pixel, logo melhor a qualidade da imagem, e portanto, maior será o seu tamanho. Uma imagem digital é guardada numa forma numérica como dados, e devido ao número de elemento da matriz necessário para termos uma imagem com boa qualidade, é bastante comum usarmos algum algoritmo de compressão para comprimirmos a imagem.

Uma imagem digital pode ser classificada como:

  • Raster ou bitmap, que é uma imagem gerada pela divisão da região em diversos pontos (um pixel), como uma matriz, e a cada ponto associa-se um vetor com informações sobre a a cor, intensidade luminosa, etc. De fato é uma matriz 3D e esse formato também é referido como um de mapa de bits (do inglês bitmap que significa mapa de bits). Note que para obtermos uma boa resolução precisamos de muitos pontos, então para armazenarmos essa matriz usamos algoritmos de compressão. Basicamente, o que difere cada um dos formatos bitmaps são os seus algoritmos de compressão, sendo uns livres e outros não. Os formatos de imagens bitmap mais comuns são
    • .bmp Formato proprietário que pode ou não ser comprimido.
    • .gif Formato proprietário ideal para imagens com poucas cores.
    • .png É um formato livre sem perdas, e bastante compacto, que foi elaborado para ser o substituto livre do .gif.
    • .jpg É um formato que usa uma compressão com perda de informação, e com isso a qualidade pode variar muito dependendo das taxas de compressão. Ele abdica de alguma da qualidade da imagem para manter os tamanhos dos arquivos pequenos. Indicado para fotos.
    • .tif É um formato proprietário, sem perdas.
  • Um desenho vetorial, em que os pontos da imagem são representados por vetores matemáticos num sistema de coordenadas cartesianas, ligados entre si por linhas que vão de um ponto a outro. Esse tipo de imagem permite um redimensionamento em qualquer escala sem perda de qualidade por ser baseada em uma estrutura matemática. Entretanto no momento da impressão o equipamento faz a sua conversão em uma imagem bitmap adequada as suas características. Note que impressoras jato de tintas, laser e outras imprimem bitmaps, pois imprimem sequências de pontos dispostos em linhas e colunas, cuja frequência, por sua vez, definem a resolução da impressora. O processo de conversão de uma imagem vetorial numa bitmap é chamado de rasterização, e é executado por um RIP (Raster Image Processor).
  • Os formatos de imagens vetoriais mais comuns são
    • .svg Formato livre, editável para gráficos vetoriais. O mais indicado para lidar com imagens vetoriais.
    • .cdr Formato proprietário da Corel(R).
    • .ai Formato proprietário do Adobe Illustrator.
    • .eps Formato livre não editável de imagens.

    • Basicamente a qualidade de uma imagem digital deve-se a dois aspectos:

    • a quantidade de pixel por polegada (resolução da imagem).
    • o número de pixels na horizontal e na vertical (tamanho da imagem em pontos ou centímetros)

    Por exemplo: Se uma imagem possuí 1000 pixels $×$ 1000 pixels, significa dizer que possuí 1 milhão de pixels, ou 1 Mp, mas isso não quer dizer necessariamente que está imagem é de alta qualidade, porque ainda falta a relação com a quantidade de pixels por polegada (DPIs) da imagem. Para uma boa definição é preciso que a imagem tenha 300 DPIs por polegada, isso permite que se faça uma cópia de alta qualidade em papel fotográfico, o tamanho desta cópia ($10 ×15$ cm ou $15 ×18$ cm) vai depender de quantos pixels a imagem possuí na vertical e quantos na horizontal.

    Primeiro vamos descrever alguns parâmetros e operadores simples que permitem que a busca seja mais eficiente. Quanto aos termos e operadores da busca temos:

    Maiúsculas/Minúsculas e Acentuação – O sistema de busca não faz distinção entre palavras que contenham letras maiúsculas/minúsculas e acentuação, portanto qualquer uma das seguintes expressões irá fornecer o mesmo resultado: FISICA, FÍsIcA, fiSiCA.

    Aspas – Com o conteúdo "entre aspas" efetua-se uma busca pela ocorrência exata da expressão que está entre as aspas, matendo a mesma sequência e forma dos termos que constituem a expressão.

    Sinal de subtração – Usa-se esse sinal/operador entre alguns termos, após a expressão de busca, para que as expressões contendo esses termos não apareçam no resultado da busca. Por exemplo, para buscar "Física" sem a palavra "Educação" deve-se digitar: física -Educação. Note que entre "física" e o sinal menos há um espaço enquanto o sinal é digitado junto com a palavra que se deseja excluir.

    Sinal de soma – Usa-se esse sinal entre alguns termos, após a expressão de busca, para que as expressões contendo exatamente esses termos apareçam no resultado da busca. Por exemplo, para buscar "Física" sem a palavra "Educação" e que contenha a palavra "Mecânica" deve-se digitar: física -Educação +Mecânica. Observe que os espaços são importantes.

    Sinal de asterisco * – Ao incluir esse caractere entre dois termos de uma busca, ele irá considerar como válida qualquer expressão entre os dois termos da busca, por isso, ele é chamado de caractere de preenchimento. Por exemplo, a busca software livre * licença irá retornar as ocorrências de software livre mais qualquer expressão/palavra e licença.

    OR ou OU Termo usado para realizar buscas alternativas. Por exemplo, para buscar Física (mecânica OU eletricidade) -automotiva -automovel -autos irá buscar todas as ocorrências de Física e mecânica sem ser automotiva ou todas as ocorrências de Física e eletricidade sem ser automotiva. Note que o termo OR ou OU precisam ser maiúsculos e que os parênteses servem para agrupar.

    define:termo – Operador para buscar por definições do termo. Por exemplo, para buscar as definições do termo piezoelétrico digite define:piezoelétrico.

    info:URL Operador que mostra informações que o Google tem sobre o sítio buscado. Por exemplo, informações sobre o sítio da UFG são obtidas digitando-se info:www.ufg.br e para obter informações do sítio do Uol digite info:www.uol.com.br. Essa busca retornará as seguintes informações para a URL:

    • Uma descrição do sítio da Web e um link para o sítio.
    • Mostrar o cache do Google para a URL.
    • Encontrar páginas na Web que sejam semelhantes a URL.
    • Encontrar páginas na Web que tenham um link para a URL.
    • Localizar páginas de sítio para a URL
    • Encontrar páginas na Web que contenham o termo "URL"

    Essas informações podem ser obtidas individualmente com um operador de busca específico.

    termo+site:URL – busca por expressões que contenham a palavra "termo" no sítio cujo endereço é URL. Por exemplo, para buscar o termo "linux" no sítio da ufg digita-se linux site:www.ufg.br.

    link:URl – operador de busca por "páginas externas" que se referenciam ao sítio URL. Por exemplo, os sítios que fazem referência ao sítio do Instituto de Física da UFG, são obtidos com link:www.if.ufg.br

    O Sistema TeX/LaTeX

    O TeX é um sistema tipografia criado por Donald E. Knuth para produção de materiais de alta qualidade tipográfica. Na realidade, ele é uma linguagem computacional que permite a definição de um sistema de composição de tipos (composição tipográfica) de alta qualidade. Sua execução permite a criação, visualização e impressão de textos técnicos especialmente aqueles que necessitam da rica simbologia matemática.

    O TeX é um processador de macros com grandes possibilidades de programação, o que torna o um sistema livre extremamente flexível, e além disso os textos resultantes desses sistema possuem uma alta qualidade gráfica. Esse sistema é excelente para criação de documentos que contenham fórmulas matemáticas, pois é muito simples escrevê-lás nele, mesmo fórmulas muito complexas, e o resultado final é excelente. Por causa disso, o TeX é extremamente popular em universidades e em meios acadêmicos.

    Junto com o TeX, Knuth também desenvolveu um conjunto de macros chamado "plain TeX" que contém um número de macros básicos para a criação de documentos. Escrever um texto diretamente em TeX, significa usar este conjunto mínimo de macros "plain TeX" e o editor de textos usado pode ser qualquer um, desde que ele salve o texto sem formatação. O uso direto do TeX não é muito comum devido à sua complexidade, pois a maioria das pessoas que utilizam o TeX o utilizam por meio do pacote de macros LaTeX, que permite a criação de documentos descrevendo o seu conteúdo, e não sua forma.

    O LaTeX/LaTeX consiste de um conjunto de macros para o processador de textos TeX, que é utilizado amplamente para a produção de textos matemáticos e científicos devido à alta qualidade tipográfica do material gerado. Atualmente há macros para variedade muito grande de textos, que vão por exemplo da edição de partituras musicais até a edição de livros de Xadrez/Gamão, Sudoku, posters, apresentações, etc. No sítio http://texcatalogue.sarovar.org/ temos o "TeX Catalogue (Preface Home Edition)" de Graham Williams, que é um catálogo bastante completo dos macros do LaTeX.

    Os macros do sistema LaTeX fornecem ao usuário um conjunto de comandos de alto nível e alta flexibilidade, que permitem abstrações para lidar com bibliografias, citações, formatos de páginas, referência cruzada e tudo mais que não seja relacionado ao conteúdo do documento em si.

    As desvantagens de usarmos o sistema LaTeX diante do processadores de textos comuns são:

    • Exige maior esforço inicial de aprendizagem se comparado aos editores de texto tradicionais.
    • Normalmente não é possível ver o resultado final do documento enquanto digitamos o texto.
    • Trabalhar com tabelas e figuras é um pouco mais difícil do que nos editores de texto tradicionais./li>
    • É muito difícil escrever documentos desorganizados e desestruturados.
    • Como não é um processador de textos específicos, não há um corretor gramatical para o documento, e dependendo do editor escolhido pode-se ou não ter um dicionário para correção do texto.

    Já algumas das vantagens em se usar o LaTeX são:

    • Os usuários precisam aprender apenas alguns poucos comandos de fácil compreensão que especificam a estrutura lógica de um documento.
    • É possível encontrar diversos "layouts" profissionalmente criados, que deixam um documento realmente com qualidade "de gráfica".
    • Torna-se extremamente fácil incluir fórmulas matemáticas complexas no texto.
    • Gera arquivos nos formatos pdf e ps livremente.
    • Possibilidade de mudar completamente a formatação do texto com um único comando.
    • É gratuito e encontrado em versões tanto para Linux como para Windows e diversos outros sistemas operacionais.
    • Até mesmo estruturas relativamente complexas como notas de rodapé, referências,índices e bibliografias podem ser criados facilmente.
    • Produção do texto é incrementada já que o autor deixa de se preocupar com detalhes de formatação.
    • A portabilidade do documento em relação aos diversos sistemas operacionais e a sua independência quanto ao editor de texto usado para escrever o documento.
    • Permite que um documento complexo com muitas páginas, seja quebrado em diversos pequenos documentos sem correr o risco de perdas e de alteração da formatação original do documento.
    • Fácil criação e manutenção de referências cruzadas, índices, glóssarios, tabelas de conteúdo, etc. Mesmo na junção de partes de um documento as referências não são perdidas.
    • Manutenção do documento, com o tempo. Os documentos escritos à vinte anos ainda podem ser trabalhados e modificados caso seja necessário, sem que haja perdas de formatação ou de texto ou fontes.
    • Portabilidade entre os diversos tipos de documentos. Um documento LaTeX escrito na forma de uma artigo pode facilmente ser transformado em um documento com uma, duas ou três colunas, em um poster, em um livro ou relatório, em uma apresentação, etc., e tudo isso sem modificações no texto original.
    • Há possibilidade de se comentar partes do texto, sem ser necessário apagar essa parte do texto do documento.

    O Gnuplot é um programa interativo livre, baseado em linhas de comando, para fazer gráficos de alta qualidade com versões para os sistemas operacionais Linux, MS Windows, Open Solaris, OSX, VMS, etc e muitos outros. Ele foi criado originalmente para permitir que cientistas e estudantes pudessem visualizar funções matemáticas e dados de modo iterativo, entretanto, ele cresceu muito e atualmente é usado por outros programas livres, tais como o octave, maxima, etc.

Ele possuí uma diversidade de drivers que permitem que ele exporte os gráfico em mais de cinquenta tipos diferentes de formatos de saída, entre os quais destacam-se: os terminas com telas interativas e a saídas para arquivos nos formatos .eps, .fig, .jpg, .svg, .pbm, .pdf, .png, LaTeX e metafont. O Gnuplot é um software pequeno e portável, o arquivo usado para gerar um gráfico pode ser usado posteriormente em uma nova versão mantendo toda sua funcionalidade.

    Toda versão do Gnuplot, possui em sua página oficial http://www.gnuplot.info/, um "link" chamado "demo gallery", que é uma galeria com uma demonstração das funcionalidades do Gnuplot, note que esta galeria é organizada por seções.     O Gnuplot é um software com as seguintes funcionalidades:

    1. Possuí um conjunto de funções matemáticas implementadas;
    2. Faz gráfico de funções algébricas, sendo eles:
      • gráficos unidimensionais;

    3. Faz gráficos de um arquivo de dados, podendo ser:
      • gráficos unidimensionais;
      • gráficos bidimensionais;
      • gráficos em escalas logarítmicas;
    4. Faz ajuste de curvas;
    5. Exporta o resultado final em mais de 50 formatos diferentes, os seus "Terminais";
    6. Permite a construção de "scripts".

    \begin{equation} f = \begin{cases} b & \text{ Se } a = 0 \\ c & \text{ Se } a \neq 0 \end{cases} \end{equation}

    Tabela 2.2 – Operadores Matemáticos
    Símbolo Exemplo Explicação
    ** a* *b Exponenciação
    * a * b Multiplicação
    / a / b Divisão
    % a%b Módulo*
    + a + b Adição
    – a – b Subtração
    Tabela 2.3 – Operador Ternário
    Símbolo Exemplo Explicação
    ?: a?b : c Operação Ternária

    Esse operador é proveniente da linguagem de programação C. A variável $a$ pode ser uma expressão inteira, então para avaliar o resultado desta expressão o primeiro argumento $a$, que deve ser um inteiro, é calculado. Se ele for verdade (diferente de zero), o segundo $b$ é calculado e retornado, de outro modo, o terceiro $c$ é calculado e retornado.

    • Em 1990 foi iniciado seu desenvolvimento pelos pesquisadores do INRIA (Institut National de Recherche en Informatique et en Automatique) e do ENPC (École Nationale des Ponts et Chaussées).
    • Em maio de 2003 foi criado o Consórcio Scilab que tornou-se responsável pelo seu desenvolvimento.
    • É distribuído gratuitamente e em código aberto via Internet desde 1994.
    • Devido sua qualidade é usado em diversos ambientes industriais e educacionais pelo mundo.

    Suas principais características são:

    • Distribuição gratuita, com código fonte disponível. Linguagem simples e de fácil aprendizado;
    • Possui um bom sistema de auxílio ao usuário, help;
    • Poderoso ambiente para geração de gráficos 2D e 3D, inclusive com animação;
    • Possui várias funções para manipulação de matrizes. As operações de concatenação, acesso e extração de elementos, transposição, adição e multiplicação de matrizes são facilmente realizadas;
    • Permite trabalhar com polinômios, funções de transferência, sistemas lineares e grafos;
    • Define-se facilmente uma função;
    • Permite o acesso a rotinas escritas nas linguagens FORTRAN ou C;
    • Processamento em paralelo;
    • Um excelente "help", o qual traz uma descrição da função desejada e um exemplo do seu uso. Além disso, ele também possuí um demo para as suas diversas habilidades.

    Introdução

    O software Maxima é um sistema para manipulação de expressões simbólicas e numéricas, incluindo diferenciação, integração, séries de Taylor, transformadas de Laplace, equações diferenciais ordinárias, sistemas de equações lineares, polinômios, conjuntos, listas, vetores, matrizes, e tensores. O Maxima produz resultados com alta precisão numérica por meio das frações exatas, inteiros com precisão arbitrária e números de ponto flutuante com precisão variável. O Maxima também pode fazer gráficos de funções e dados em duas e três dimensões.

    O Maxima é um sistema de código aberto que pode ser compilado em muitos sistemas operacionias, incluindo Windows, Linux e MacOS X. Em sua página oficial, o código está disponibilizado, assim como os binários compilados para Windows e Linux.

    Atual versão do Maxima é uma evolução do sistema de computação algébrica Macsyma desenvolvido no fim dos anos 1960 pelo ``Massachusetts Institute of Technology'' (MIT). Em 1982 o MIT forneceu uma cópia do Macsyma para o Departamento de Energia (DOE) e essa cópia, que passou a ser chamada de Macsyma DOE, foi mantida pelo professor William F. Schelter da Universidade do Texas no perído de 1982 à 2001, ano de sua morte. Além disso, em 1998, o professor Schelter obteve permissão do Departamento de Energia para liberar o código fonte do Macsyma DOE sob a Licença Pública GNU, e em 2000 ele iniciou o projeto Maxima no SourceForge para manter e desenvolver o Macsyma DOE, que desde então passou a ser chamado Maxima.

    A versão do Maxima sobre a qual este trabalho irá tratar é a 5.21.1, a qual é a que está disponível na última versão do ubuntu 10.10. A versão mais atual, entretanto é a 5.23.2.

    Operações Básicas

    Ao iniciar o Maxima abre uma interface de terminal na qual aparece o símbolo (%i1)representa a posição na memória de cada operação a ser realizada.

    As operações serão digitadas na frente do símbolo (%i1), em que o "i" no meio significa entrada do inglês "INPUT", enquanto a saída do programa será representada pelo símbolo (%ox), com o "o" sendo saída do inglês "OUTPUT".

    Toda expresão digitada deve terminar com um ";"" para indicar ao Maxima que a expressão já pode ser avaliada, como no exemplo abaixo:

    (%i1) 1 + 2; (ENTRADA - INPUT - representado pelo i no Maxima)
(%o1) 3 (SAÍDA - OUTPUT - representada pelo o no Maxima)

    Funções

    É muito simples definir funções no Maxima, é muito parecido com o modo normal de se escrever, mudando somente o símbolo de atribuição que no caso é :=, como pode se observar o exemplo:

    (%i1) f(x):=x+2;
(%o1) f(x) := x + 2
(%i2) f(5);
(%o2) 7

    Pode-se definir também funções de n variáveis, respeitando a mesma sintaxe da anterior, somente colocando as variáveis entre vírgulas, do seguinte modo:

    (%i1) g(x,y,z) := x * y + 2 * z;
(%o1) g(x, y, z) := x y + 2 z
(%i2) g(1,2,3);
(%o2) 8

    Limites

    Limites são facilmente interpretados pelo Maxima, e possui uma sintaxe bastante simples, limit(função, variável, valor que tende a variável);. Por exemplo:

    (%i1) limit( (2*x+1)/(3*x+2), x,inf );

(%o1) $ \frac{2}{3} $

(%i1) limit(((z^(2/3))/(z - sqrt(2 * z))), z, 8);

(%o1)  1

(%i2) limit((t^2 + 6*t + 9)/(9 - t^2), t, -3);

(%o2) 0

    Alguns exemplos para funções trigonométricas:

    (%i1) limit(sin(x)/x, x, 0);

(%o1)  1

(%i2) limit((1 - cos(3*x))/(2*x^2), x, 0);

(%o2)  $\frac{9}{4}$

(%i3) limit((1 - cos(3*x))/(3*x), x, 0);

(%o3)  0

    Agora vamos tratar com os limites laterais. Para obter os limites de uma função quando seu argumento tende a um dado valor pela esquerda, basta adicionar um quarto parâmetro, o minus:

    (%i1) limit(sqrt(x * (5-x)), x , 5, minus);
(%o1) 0
(%i1) limit(1/x, x, 0, minus) ;
(%o1) minf

    Já ao tomar o limite pela direita, basta adicionar o quarto parâmetro plus:

    (%i1) limit(sqrt((4*x)/(x-4)), x, 4, plus);
(%o1) inf
(%i2) limit(1/x, x, 0, plus);
(%o2) inf

    Como a derivada de uma função $f(x)$ é definida pelo seguinte limite,

    \begin{equation} f'(x) = \lim_{\Delta x \to 0} \frac{f(x + \Delta x) - f(x) )}{\Delta x} \end{equation} 4.3

    então, pode-se obter a derivada de uma função, utilizando apenas limites. Considere por exemplo o caso em que $ g(x) = f '(x) $, e $f(x)=2x^2 + 3x$ conforme mostrado abaixo:

    (%i1) f(x):= 2 * x^2 + 3 * x;

(%o1) f(x) := 2 x^2 + 3 x

(%i2) g(x) = limit((f(x + h)- f(x)) / h, h, 0);

(%o2) g(x) = 4 x + 3

    Um outro exemplo:

    (%i1) f(x):=x - 1/x;

(%o1) $f(x) = x - \frac{1}{x}$

(%i2) g(x) = limit((f(x + h)- f(x)) / h, h, 0);

(%o2) $g(x) = \frac{{x}^{2}+1}{{x}^{2}}$

    Para o cálculo de um limite, o Maxima possui uma variável nomeada de lhospitallim, a qual designa o número máximo de vezes que a regra L'Hospital é usada em um cálculo de limite. Isso evita ciclos infinitos em casos como limit(cot(x)/csc(x), x, 0);. O valor padrão no Maxima é 4.

    Assim, visto como implementar limites no Maxima, passaremos então as derivadas.

    Derivadas

    A diferenciação de uma função pode ser rapidamente obtida pela função diff, cuja sintaxe é a seguinte diff(função, variável);. No exemplo

    Aplicação:

    (%i1) diff((2*x^2+3*x), x);

(%o1) $ 4 x + 3 $

(%i2) diff((x - 1/x), x);

(%o2) $\frac{1}{x^2} + 1$

(%i3) diff(sin(x), x);

(%o3) $cos(x)$

    \begin{equation} f(x) = 2cos(x^3) + \dfrac{sin(3x^2)}{x^2} + 4 \ln(x^{7/2}) \end{equation} 4.4 

    (%i1) f(x) := 2*cos(x^3)+sin(3*x^2)/(x^2)+4*log(x^(7/2));

       +4\,\mathrm{log}\left( {x}^{\frac{7}{2}}\right)$

$\frac{540\sin(3\,{x}^{2})}{{x}^{2}}+\frac{120\sin(3\,{x}^{2})}{{x}^{6}}-\frac{360\cos(3\,{x}^{2})}{{x}^{4}}+432\cos(3\,{x}^{2})-\frac{84}{{x}^{4}}$

    Integrais

    Para integrar uma função usa-se a função integrate, cuja sintaxe para integrais indefinidas é integrate(função, variável); enquanto para para integrais com limites definidos a sintaxe é integrate(função, variável, limite_inferior, limite_superior);.

    Considere os seguintes exemplos de integrais indefinidas:

    (%i1) integrate(x^3*((1 + x^4)^5), x);

(%o1) $\frac{{\left( {x}^{4}+1\right) }^{6}}{24}$

(%i2) integrate(x^2*cos(4*x^3), x);

(%o2) $\frac{\sin\left( 4\,{x}^{3}\right) }{12}$

(%i3) integrate(1/(x^2+a^2), x);

(%o3) $\frac{\mathrm{atan}\left( \frac{x}{a}\right) }{a}$

    Considere os seguintes exemplos de integrais definidas:

    (%i1) integrate((1+sqrt(t))^2/sqrt(t), t, 1, 4);

(%o1) $\quad \frac{38}{3} $

(%i2) integrate(6 - x^2 -x, x, -3, 2);

(%o2) $\quad \frac{125}{6} $

(%i3) assume(a>0);  /* Dizemos que a>0 */

(%o3) $\quad [a>0]$

(%i4) integrate(1/(x^2+a^2), x, 0, inf);

(%o4) $\quad \frac{\pi}{2\,a}$

(%i5) integrate(sin(2*x)*tan(2*x), x,0,%pi/8);

(%o5) $\quad \frac{\frac{\mathrm{log}\left( \frac{\sqrt{2}+2}{2}\right) }{2}-\frac{\mathrm{log}\left( -\frac{\sqrt{2}-2}{2}\right) }{2}-\frac{1}{\sqrt{2}}}{2}$

(%i6) ratsimp(%);} /* Irá simplificar o resultado anterior */

(%o6) $\quad \frac{\sqrt{2}\,\mathrm{log}\left( \frac{\sqrt{2}+2}{2}\right) -\sqrt{2}\,\mathrm{log}\left( -\frac{\sqrt{2}-2}{2}\right) -2}{{2}^{\frac{5}{2}}}$

    (%i1) integrate(1/x,x,-1,0);
defint: integral is divergent.

    Nesse caso utilize os recursos de limites para verificar a condiçãos; de continuidade da função.

    Construção de Gráficos

    A seguir serão apresentadas as funções mais usadas para a construção de gráficos com um exemplo de sua utilização.

    O comando plot2d(expressão, x_range, …, opções, …); faz o gráfico da expressão como uma função da variável nomeada em x_range, sobre o intervalo especificado em x_range. Se a amplitude vertical não for alternativamente especificada por set_plot_option, essa é escolhida automaticamente. Todas as opções possuem um valor padrão a menos que sejam alternativamente especificadas pelo comando set_plot_option. Vejamos por exemplo,

    (%i1)  plot2d(cos(x), [x, -4, 4])＄

    Agora vamos interpretar este comando. A função cujo gráfico será feito é o cos(x). O primeiro parâmetro do termo [x, -4, 4], significa que a variável do gráfico unidimensional é o x, e os outros dois valores representam o valor mínimo e máximo que a variável x irá assumir no gráfico. Já no próximo exemplo, adicionamos os limites de variação do eixo das ordenadas, o eixo $y$.

    (%i1)  plot2d (sec(x), [x, -2, 2], [y, -20, 20])＄

    ou ainda

    (%i1) plot2d(tan(x), [x, -2, 2], [y, -18, 18]);

    Podemos ainda fazer o gráfico de duas ou mais funções juntas, em um mesmo intervalo de variação, conforme ilustra o exemplo a seguir.

    (%i1) h(x) := if x = 0.0 then 1 else sin(x)/x＄
(%i2) f(x) := sin(x)/x＄
(%i3) g(x) :=  exp(-0.5*x^2)＄
(%i4) plot2d([h,g], [x, -20, 20])＄
(%i5) plot2d([sin(x)/x, g], [x, -20, 20])＄
plot2d: expression evaluates to non-numeric value somewhere in plotting range.
