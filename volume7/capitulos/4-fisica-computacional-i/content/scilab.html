Scilab

Introdução

O Scilab é um software científico para computação numérica bastante flexível e que fornece um poderoso ambiente computacional de código aberto para aplicações científicas e em engenharia. Sua origem e desenvolvimento podem ser sumarizados em:

  • Em 1990 foi iniciado seu desenvolvimento pelos pesquisadores do INRIA (Institut National de Recherche en Informatique et en Automatique) e do ENPC (École Nationale des Ponts et Chaussées).
  • Em maio de 2003 foi criado o Consórcio Scilab que tornou-se responsável pelo seu desenvolvimento.
  • É distribuído gratuitamente e em código aberto via Internet desde 1994.
  • Devido sua qualidade é usado em diversos ambientes industriais e educacionais pelo mundo.
83

Suas principais características são:

  • Distribuição gratuita, com código fonte disponível. Linguagem simples e de fácil aprendizado;
  • Possui um bom sistema de auxílio ao usuário, help;
  • Poderoso ambiente para geração de gráficos 2D e 3D, inclusive com animação;
  • Possui várias funções para manipulação de matrizes. As operações de concatenação, acesso e extração de elementos, transposição, adição e multiplicação de matrizes são facilmente realizadas;
  • Permite trabalhar com polinômios, funções de transferência, sistemas lineares e grafos;
  • Define-se facilmente uma função;
  • Permite o acesso a rotinas escritas nas linguagens FORTRAN ou C;
  • Processamento em paralelo;
  • Um excelente "help", o qual traz uma descrição da função desejada e um exemplo do seu uso. Além disso, ele também possuí um demo para as suas diversas habilidades.

Iniciando com o Scilab

Agora uma vez na interface do Scilab, digite os comandos abaixo e veja o que ocorre.

-->a=1    // Verifique o que ocorre

-->a      // Veja o que ocorre
-->clear  // Veja o que ocorre
-->a      // Veja o que ocorre

-->i=1:5      // Verifique o que ocorre
-->i=1:0.5:5  // Veja o que ocorre

-->a=2,b=3   // Veja o que ocorre
-->clear     // Reinicializa as variáveis do sistema
-->a
-->b

De acordo com o exemplo, anterior o sinal de ; no fim de um comando diz para o Scilab executar o comando sem fornecer um retorno.

-->a=5,b=10;
-->save('dados.dat',a,b) // Salva no arquivo binário as variáveis a e b
-->clear                 // Limpa as variáveis
-->a
-->b
-->load('dados.dat','a','b') // Lê as variáveis a e b no arquivo binário
-->a
-->b
-->d1=diary('file.txt')//Salva seção, o que for digitado a partir deste ponto
-->a=10,v0=0,x0=10;
-->t=1;
-->x=x0+v0*t+0.5*a*t^2;
-->diary(d1,'close')          // Fecha o arquivo.

A função diary é muito útil para testarmos algumas funções ou a digitarmos alguns valores iniciais, pois ela guarda aquilo que digitamos em um arquivo que pode ser lido posteriormente.

No exemplo abaixo, apresentamos um outro comando que é bastante útil quando estamos fazendo alguma operação e precisamos de calcular um outro valor antes de prosseguirmos.

-->a=2, pause
    a  =
          2.
-1->a=3
     a  =
          3.
-1->a=2*a
     a  =
          6.
-1->return
-->a
    a  =
        2.

O comando pause, interrompe a execução da seção atual e abre uma nova seção, que recebe um prompt para indicar o nível de pause (exemplo: -1->). Neste ponto, o usuário está num novo espaço de trabalho em que todas as variáveis de nível inferior (em particular todas as variáveis da função) estão disponíveis. Para retornar ao espaço de trabalho em que o comando pause foi chamado, usa-se o comando "return" Neste modo, […]=return(…) retorna as variáveis dos argumentos (…) ao espaço de trabalho inicial com nomes na saída […]. Caso contrário, as variáveis de nível inferior são protegidas e não podem ser modificadas. Este modo pode ser cancelado pelo comando "abort".

Polinômios

O Scilab possuí uma função interna específica para tratar com polinômios. No Scilab os polinômios

  • são criados através da função poly.
  • de mesma variável, podem ser somados, subtraídos, multiplicados e divididos.
  • podem ser gerados através de suas raízes.
  • podem ser gerados através de seus coeficientes.

Para um vetor A, p=poly(A,'x') define o polinômio p(x) com a variável x de modo que roots(p) = A).

-->A=[-2 3 7];   // Cria um vetor
-->p=poly(A,'x') //Cria um polinômio cujas asa raízes são o vetor de entrada.
p  =
               2   3
    42 + x - 8x + x
-->roots(p)  // Determina as raízes do polinômio
 ans  =
    3.
  - 2.
    7.

Por exemplo, verifique cada um dos exemplos abaixo:

  • O comando p=poly([-1,2],'x') cria um polinômio com raízes $-1$ e $2$.
  • O comando roots(p) encontra as raízes do polinômio p.
  • O comando q=poly([-4 2 -1], 'x', 'coeff') cria o polinômio $q(x)= - 4 + 2x - x^{2}$.
  • Podemos somar, subtrair multiplicar e dividir polinômios.

Matrizes e elementos de Matrizes

Vamos considerar as seguintes matrizes

\[ A= \begin{pmatrix} 1 & 2 & 3 \\ -5 & 4 & -2 \\ -3 & 7 & 1 \end{pmatrix} \qquad B= \begin{pmatrix} -3 & 4 & -1 \\ 4 & -2 & 8 \\ 2 & -5 & 1 \end{pmatrix} \]

O comando:

  • A = [1 2 3; -5 4 -2; -3 7 1] cria a matriz $A$.
  • B = [-3 4 -1; 4 -2 8; 2 -5 1] cria a matriz $B$
  • A(1,2) mostra o elemento $A_{1,2}$ da matriz $A$.
  • size(A) é uma matriz de $3 \times 3$.
  • A' é a transposta da matriz A.
  • V(2:5) acessa todos os elementos $V_{2}$, $V_{3}$, $V_{4}$ e $V_{5}$ do vetor $V$.
  • V(1:2:9) acessa todos os elementos ímpares entre $1$ e $9$ do vetor $V$.
  • A(:,2) lista todos os elementos da coluna $2$.
  • v(\$), acessa último elemento de v.
  • A(\$-1,:), acessa a segunda linha da matriz $A$.
  • testmatrix gera algumas matrizes particulares.
  • testmatrix('hilb',n), gera uma matriz de Hilbert de dimensão n.
  • testmatrix('magic',n), gera uma matriz quadrado mágico de dimensão n.
  • trace(M) retorna o traço da matriz M
  • B=rand(n,m) ou B=rand(A), gera matrizes com elementos aleatórios.
  • A(1,:) lista todos elementos da linha 1 enquanto
  • A(:,2) lista todos elementos da coluna 2.
  • v=[A(1,:); A(3,:)] cria um vetor coluna:
  • sum(B,'r') e prod(B,'r') respectivamente somam e fazem o produto por colunas, para produzir uma linha.
  • sum(B,'c') e prod(B,'c') respectivamente somam e fazem o produto por linhas, para produzir uma coluna.
  • [x0,nsA] = linsolve(A,c[,x0]), determina a solução do Sistema Linear $A \cdot x+c = 0$:

Gráficos

Vamos iniciar fazendo gráficos simples, conforme o exemplo:

-> x=1:10;  // Cria vetor x de 10 elementos,com passo 1
-> plot(x,x^2,'b-',x,10*x,'r-'); // Note: 'b-' é azul com traço

O comando clf() apaga o gráfico corrente. então

-> clf(); // apaga o gráfico anterior
-> t=-2*%pi:%pi/100:2*%pi;plot(t,sin(t),’r-’);
-> clf(); // apaga o gráfico anterior
x=linspace(1,11,20);
y=linspace(-2,2,20);
plot2d(x,y,style=-1);

Para fazer gráficos no Scilab, primeiro é necessário definir uma função. No Scilab isso é feito usando o comando deff da seguinte forma

deff('[y1]=f1(x)','y1=x^2-2*x-3'); // Uma variável
deff('[y2]=f2(x)','y2=x^2-2');            // Uma variável
[s1]=fsolve(1,f,0.0001);   // Determina as raízes de f1(x)
[s2]=fsolve(1,f,0.0001);   // Determina as raízes de f2(x)
x=-5:0.1:5;                // Cria o vetor x com o intervalo
plot(x,f1);                // Faz o gráfico de f1
plot(x,f2);                // Faz o gráfico de f2
deff ('[y]=f(x)','y=3*x+x.*sin(x)')
x=1:.5:100;
fplot2d(x,f)
