Parágrafos

Esse é um exemplo de parágrafo. Todo esse texto está envolvido em uma tag <p></p>. Para começar um novo parágrafo, você deve abrir uma nova tag, logo abaixo dessa. Não adianta quebrar linha usando a tecla ENTER no editor de código, o navegador vai ignorar tudo e colocar todo o texto quebrado em uma linha só. Ele também ignora excesso de espaços. Quando você está escrevendo um documento HTML, quebrar linhas é interessante apenas para você organizar seu próprio código. Para diagramação, é usado o a linguagem CSS.

Esse texto começou em um novo parágrafo. Perceba o espaço entre esse texto e o de cima. Esse espaço existe por causa do código CSS que eu escrevi, não porque eu quebrei a linha entre esse parágrafo e o anterior.

Tanto que esse parágrafo foi escrito na mesma linha que o parágrafo anterior, e mesmo assim, existe espaço entre eles.

Se deseja quebrar o texto dentro de um parágrafo, use a tag <br>. Onde houver ocorrências dessa tag, a linha será quebrada logo após ela. Essa tag não pode possuir conteúdo textual, portanto, ela não "abre e fecha" como as outras tags, não é necessário colocar um </br> depois do <br>.

Tags de título (headings)

As tags de título são as tags <h1>, <h2>, <h3>, <h4>, <h5> e <h6>. Essa numeração de cada uma indica o peso desse cabeçalho. Isto é, o <h1> é a tag de título de maior hierarquia, enquanto o <h2> indicam uma seção referente ao <h1>, enquanto o <h3> é uma subseção do <h2>, e assim por diante.

O <h1> geralmente é usado para o título do material todo, portanto, ele aparece uma única vez, no topo do material. Todas as seções e subseções desse material usam o <h2> em diante.

Para deixar mais evidente essa hierarquia, as tags de título de 2 à 4 são enumeradas automaticamente. Veja como é no exemplo abaixo:

Links externos

Assim como qualquer site, um trecho de texto pode ser também um link para um site externo. Para isso, usa-se a tag <a>. Porém, para definir para qual link essa tag leva, você precisa definir esse link no "atributo" href. Esse atributo fica dentro da tag de abertura, e não entre as tags de abertura e fechamento. Para evitar erros, é só copiar essa tag <a> que usei nesse parágrafo como exemplo.

Apesar do texto entre a abertura e o fechamento poder ser qualquer texto, como nesse exemplo de link que leva para o Facebook, nesse e-book, tanto o atributo href quando o texto da tag devem o próprio link. Por exemplo, um link que leve para o site do CIAR deve ser assim:

http://www.ciar.ufg.br (O texto da tag é também o próprio link)

E não assim:

Site do CIAR (O texto da tag é outra coisa)

As tags <a> devem ter 2 atributos:

  • href: O link do site externo.
  • target: Deve sempre ser target='blank', para o link abrir uma nova aba do navegador.

Imagens e fotos

Para imagens e fotos, usa-se a tag <figure>. Dentro dela, devem haver 2 tags:

  • <img> é a tag que possui a imagem de fato. Todas as imagens usadas no material ficarão na pasta media. O caminho da imagem é definido pelo atributo src. Essa tag possui uma peculiaridade: Ela não "abre e fecha" como as tags comuns, porque essa tag não pode conter nenhum conteúdo textual. Portanto, não existe um </img> fechando a tag. A tag deve ficar assim: <img src="media/1/teste.jpg">, e nada a mais do que isso.
  • <figcaption> é a tag com a descrição da foto. Essa é uma tag normal de texto. Cada imagem deve ter apenas uma tag <figcaption>, portanto, se a descrição da imagem precisar quebrar a linha, use a tag <br>.

Nada mais deve constar dentro da tag <figure>, apenas essas 2 tags e seus devidos conteúdos.

Tabelas

Tabelas em HTML são mais complexas que as outras, pois cada célula da tabela é uma tag. As tags usadas para tabelas são as seguintes:

  • <table>: é a tag que engloba toda a tabela. Serve apenas para delimitar o começo e o fim de uma tabela.
  • <thead>: Engloba as células que serão o "cabeçalho" da tabela.
  • <tbody>: Engloba as células que serão o conteúdo da tabela.
  • <tr>: As tabelas são divididas em linhas, depois em colunas. Essa tag é quem engloba as células que representam as colunas.
  • <th> e <td>: Essas são as tags que representam as células individuais. o <th> representa células de cabeçalho, recebendo mais destaque que o <td>, que são as células comuns.
  • <caption>: É a legenda da tabela. Funciona exatamente como a tag de legenda das imagens, isto é, só pode haver uma tag dessa por tabela, e se quiser multiplas linhas nesse legenda, use a tag <br>.

Fórmulas matemáticas

O grande diferencial desse ebook é que ele possui MUITAS fórmulas matemáticas. Para nosso facilidade, o próprio navegador consegue gerar essas fórmulas, através de um código que a gente consegue digitar como um texto comum. Essas fórmulas estão todas nos arquivos LaTeX do livro impresso.

Existem, basicamente, 2 tipos de fórmula: As fórmulas em linha e em bloco.

Fórmulas em linha se encontram no meio do texto, assim:

Sejam os vetores: $\vec{A}=(2,\beta ,1)$, $\vec{B}=(5,4,\gamma )$ e $\vec{C}=(\alpha ,4,2)$. Se $\vec{C}=\vec{A}+\vec{B}$, quais os valores de $\alpha $, $\beta $ e $\gamma $?

E as fórmulas em bloco ficam assim:

\begin{equation} \vec{S}=\vec{A}+\vec{B}=(\underbrace{A_{x} +B_{x} }_{S_{x} },\underbrace{A_{y} +B_{y} }_{S_{y} },\underbrace{A_{z} +B_{z} }_{S_{z} }), \end{equation} 1

As instruções de como aplicar essas fórmulas estão no Docs.

Enquanto às fórmulas em linha são colocadas como um texto comum no parágrafo, as fórmulas em bloco devem ficar em uma nova tag de parágrafo, separada do texto. E esse parágrafo vai ser um pouco diferente dos outros:

  • Esse parágrafo deve receber a classe formula, ficando, então, <p class="formula"></p>
  • Dentro dessa tag, vai a fórmula em bloco.
  • Algumas fórmulas são numeradas, para fins de referência. Essa numeração é uma tag <span class="numero_formula">1</span>, e deve vir depois do texto da fórmula.

Referências às fórmulas

Você encontrará, eventualmente, no meio dos textos, referências às fórmulas através do número, por exemplo Veja a fórmula 1.3. Essas referências, no material em Latex, usam um código para fazer a referência automaticamente através das tags em Latex ref e eqref. Essas tag devem ser apagadas e substituídas pela referência digitada, mesmo, sem automatização.
